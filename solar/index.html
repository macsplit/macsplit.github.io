Operational logic

How the proposal works

Inputs

  • Spreadsheet upload with solar strength, cloud attenuation, and daily output
  • Array metadata such as angle, capacity, efficiency, and location
  • Optional API connection for future automated collection

Processing

  • Normalize daily data across multiple related arrays
  • Model expected yield using clear-sky and satellite-informed context
  • Detect sustained deviations and classify probable fault types

Outputs

  • Fault type, affected array, date window, and confidence score
  • Health report summarizing degradation, warnings, and validation needs
  • Plain language guidance for manual technical follow-up
